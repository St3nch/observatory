# M20 Known Limits and Deferred Capabilities

Status: acceptance companion; owner acceptance required with v1 verdict
Milestone: M20
Date: 2026-07-12

## Purpose

Prevent bounded proof results from being mistaken for production readiness or silent feature authorization.

## Proven limits

### Local and bounded

Observatory v1 is a local repository-based proof system. It is not deployed and has no production service-level guarantees.

### No live persistence layer

No live Postgres database, physical schema, migrations, object store, or production raw archive exists. Logical planning and local proof structures do not equal production persistence authority.

### Provider ceiling

Only one narrowly bounded DataForSEO probe was executed. No additional endpoint, provider, purchase, recurring capture, Ahrefs, Semrush, marketplace, video, or AI-answer-surface execution is admitted.

### Typed-read ceiling

Typed reads are local fixture/in-memory proofs. No production API/MCP, authentication, authorization, rate limiting, network deployment, or consumer integration has been proven.

### Consumer ceiling

SearchClarity usefulness is demonstrated as a proof workflow only. Observatory does not generate customer reports, manage engagements, store orders, hold customer records, or promote conclusions automatically.

### Overlay ceiling

Overlay proof is synthetic and code-path bounded. It does not prove secure memory wiping, operating-system swap behavior, observability-platform logging, production request isolation, or real private telemetry handling.

### Recovery ceiling

Repository archive and restore are proven. Encrypted backup, off-machine storage, multiple verified generations, cloud recovery, workstation recovery, database recovery, and production disaster recovery are not proven.

### Recurrence ceiling

Recurring capture execution is explicitly rejected for v1. No scheduler, cron task, Windows task, queue worker, automatic retry, adaptive cadence, or autonomous spend exists.

## Deferred capabilities

The following remain deferred or forbidden until a future owner-approved roadmap explicitly activates them:

```text
production deployment
Postgres creation and physical schema
schema migrations
production raw archive/object storage
additional provider admission or paid pulls
Ahrefs/Semrush integration
marketplace/video/AI-answer-surface execution
recurring capture and scheduling
production API/MCP
consumer authentication and authorization
customer data or first-party analytics storage
customer-facing report generation
dashboard/operator console
cross-scope aggregation
embeddings/vector store
strategy/recommendation storage
automatic conclusion promotion
encrypted/off-machine backup and recovery
automatic backup jobs
destructive cleanup of preserved proof artifacts
```

Any unresolved owner-ruling item remains fail-closed. V1 acceptance does not resolve unrelated deferred rulings by implication.
